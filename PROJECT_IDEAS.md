# Nitpick Project Ideas Queue

This queue stores ideas for future `nitpick-packages` modules to tackle one by one.

## Pending Packages

1. **`nitpick-vdom` (Virtual DOM)**: A library to build reactive, component-based UI trees (similar to React). It can serve as the foundational architecture for building interactive front-end applications, diffing states to generate patches or re-rendering UI trees efficiently.
2. **`nitpick-graphql`**: A robust GraphQL server implementation to parse query syntax, resolve data fetching dynamically, and serve structured JSON responses. Great pairing with `nitpick-express`.
3. **`nitpick-oauth2`**: A streamlined package for handling OAuth 2.0 and OpenID Connect flows, making it trivial to add "Login with Google/GitHub" securely to any Nitpick web app.
4. **`nitpick-physics2d`**: A native 2D physics engine handling collisions, gravity, and rigid body dynamics. Useful for integrating with `nitpick-raylib` and `nitpick-sdl2` games.
5. **`nitpick-pdf`**: A library to programmatically generate, construct, and export PDF documents, including text layout and graphics drawing.
6. **`nitpick-wasm-bindgen`**: A bridging tool to seamlessly compile Nitpick code into WebAssembly (WASM) and interoperate deeply with JavaScript APIs.
